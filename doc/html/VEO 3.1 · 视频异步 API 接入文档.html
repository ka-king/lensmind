VEO 3.1 · 视频异步 API 接入文档

通过 POST /v1/videos 创建 8 秒视频任务，支持文生视频、首尾帧图生视频、多参考图生成视频。

! 内容审查避坑提醒：提交 VEO 3.1 任务前建议先看详细指南 如果返回 didn't pass content review，重试或换账号通常无效，需要改提示词、换参考图或避开受保护 IP。 先排查真人正脸、名人、未成年人 再检查性/暴力/危险、医疗病症、政治宗教 多参考图和首尾帧都要逐张确认素材风险 查看详细页面 →

提交任务

curl -X POST https://xxx.xxx.xxx/v1/videos \
  -H "Authorization: Bearer api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "veo-3-1",
    "prompt": "a cinematic product reveal, smooth camera movement, natural lighting",
    "duration": 8,
    "aspect_ratio": "16:9"
  }'

图生视频

curl -X POST https://xxx.xxx.xxx/v1/videos \
  -H "Authorization: Bearer api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "veo-3-1",
    "prompt": "the subject turns toward the camera, cinematic lighting",
    "duration": 8,
    "aspect_ratio": "9:16",
    "images": [
      "https://your-cdn.com/first-frame.jpg"
    ]
  }'

首尾帧模式

curl -X POST https://xxx.xxx.xxx/v1/videos \
  -H "Authorization: Bearer api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "veo-3-1",
    "prompt": "animate from the first frame to the last frame with smooth natural motion",
    "duration": 8,
    "aspect_ratio": "16:9",
    "images": [
      "https://your-cdn.com/first-frame.jpg",
      "https://your-cdn.com/last-frame.jpg"
    ]
  }'

多参考图

curl -X POST https://xxx.xxx.xxx/v1/videos \
  -H "Authorization: Bearer api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "veo-3-1",
    "prompt": "combine these references into one polished brand video",
    "duration": 8,
    "aspect_ratio": "16:9",
    "Ingredients_images": [
      "https://your-cdn.com/ref-1.jpg",
      "https://your-cdn.com/ref-2.jpg",
      "https://your-cdn.com/ref-3.jpg"
    ]
  }'

字段

字段类型必填说明
modelstring是固定为 veo-3-1
promptstring是视频内容描述
durationint是固定为 8，其他值会返回 400
aspect_ratiostring否16:9 / 9:16，默认横屏
imagesarray否图生视频参考图，传 1 张为单图生视频，传 2 张为首尾帧模式；顺序为首帧、尾帧。数组项使用图片 URL 字符串
Ingredients_imagesarray否多参考图模式，最多 8 张；传入后优先按多参考图视频生成处理

查询返回字段

字段说明
id / task_id任务 ID，用于继续查询
status / state / task_status任务状态，三者值一致，便于兼容不同中转站
progress进度，0-100
video_url视频任务完成后的主结果地址
url完成后的通用结果地址，视频任务等同于 video_url
metadata.result_urls结果地址数组，通常取第一项即可
seconds / duration任务时长，来自提交参数
aspect_ratio画面比例，来自提交参数
successstatus 为 completed 时为 true
finalcompleted 或 failed 时为 true
error.message失败原因，仅失败时返回

返回状态

status含义说明
queued排队中继续轮询
in_progress生成中继续轮询
completed已完成读取 video_url、url 或 metadata.result_urls
failed失败查看 error.message

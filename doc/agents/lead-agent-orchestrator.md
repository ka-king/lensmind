# Lead Agent Orchestrator — 设计文档

## 1. 角色定位

Lead Agent（电商视频主编）是 LensMind 的核心编排器。它不直接执行视频生成操作，而是将任务拆解后**委托给 6 个专业子 Agent**，并监督每一步的执行质量。

## 2. 编排流程

```
用户输入
    │
    ▼
┌─────────────────────────────────┐
│ Lead Agent: 电商视频主编          │
│                                  │
│ 1. 理解需求，澄清模糊点            │
│ 2. 委托 → 产品分析师              │
│ 3. 审查分析结果，决策风格          │
│ 4. 委托 → 编剧（生成分镜脚本）     │
│ 5. 审查脚本，确认分镜数/时长       │
│ 6. 并行委托 → 模特图生成师         │
│              → 场景设计师          │
│ 7. 逐镜委托 → 分镜动画师           │
│ 8. 委托 → 剪辑师（合成+配音）      │
│ 9. 交付最终视频                    │
└─────────────────────────────────┘
```

## 3. 安全模型

参考 DeerFlow 的 maintainer-orchestrator 设计：

- **只读决策** — Lead Agent 审查子 Agent 输出后才进入下一步，不盲目信任
- **委托边界** — 每次委托是独立的子 Agent 运行，有明确的输入/输出契约
- **可回滚** — 任何一个子 Agent 失败，可以重试而不影响已完成步骤

## 4. 澄清机制

当用户需求不明确时，Lead Agent 通过 `ClarificationMiddleware` 反问：

| 场景 | 澄清问题 |
|------|---------|
| 产品名模糊 | "这个产品是哪个品类？请补充描述" |
| 风格未指定 | "你偏好快节奏带货还是温柔叙事？" |
| 图片不足 | "能否再上传一张产品细节图？" |
| 时长不合理 | "30 秒对于 10 个卖点的产品偏短，建议 60 秒，可以吗？" |

## 5. 质量门控

每一步委托都有质量检查：

```
ProductAnalysis → 至少 3 个卖点，有目标人群 → 通过
VideoScript     → 每个分镜有口播+画面+运镜 → 通过
ModelImage[]    → 图片张数 = 分镜数，模特一致 → 通过
SceneImage[]    → 图片张数 = 分镜数，场景匹配 → 通过
VideoClip[]     → 片段数 = 分镜数，时长匹配 → 通过
FinalVideo      → 文件存在，可播放 → 通过
```

## 6. 主编 Prompt 设计

```
你是一个专业的电商视频制作主编。你的团队有 6 位专家：
- 产品分析师：擅长提炼产品卖点
- 编剧：擅长创作带货口播和分镜
- 模特图生成师：擅长生成产品展示模特图
- 场景设计师：擅长设计视频背景
- 分镜动画师：擅长将静态图转为动态视频
- 剪辑师：擅长配音、字幕和最终合成

你的职责是：
1. 理解用户需求
2. 把任务拆解后委托给合适的专家
3. 审查每步结果，确保质量
4. 把控整体进度和最终交付
```
